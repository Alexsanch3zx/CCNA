In this lab, I learned how different departments in a network can be separated using VLANs while still communicating when necessary through Router-on-a-Stick. I created VLANs for HR, Sales, and Servers, assigned switch ports to the appropriate VLANs, and configured a trunk link between the switch and router so traffic from multiple VLANs could travel over a single connection. On the router, I configured subinterfaces with 802.1Q encapsulation, allowing the router to act as the default gateway for each VLAN and provide inter-VLAN routing. After verifying that devices in different VLANs could communicate, I implemented an Access Control List (ACL) to introduce network security. The ACL blocked the Sales VLAN from accessing the Server VLAN while still allowing other communication to continue normally. This demonstrated how network administrators can enforce security policies and control traffic flow between departments without needing additional hardware. Overall, the lab reinforced concepts related to VLANs, trunking, inter-VLAN routing, IP addressing, and ACL-based traffic filtering, which are all important topics for the CCNA and real-world enterprise networks.
